# /hk:pre-clear:resume

이전 세션이 `/hk:pre-clear:save`로 남긴 핸드오프 파일을 읽어 컨텍스트를 복원한다. 복원이 끝나면 핸드오프 파일을 **자동 삭제**한다 (1회성 정책).

- 읽기 대상: `.hk/pre-clear/handoff.md`
- 삭제 시점: 사용자에게 복원 보고가 끝난 직후
- 짝 명령: `/hk:pre-clear:save`

## 실행 절차

### 1) 핸드오프 파일 존재 확인

```bash
test -f .hk/pre-clear/handoff.md
```

- **없음** → 다음 메시지 출력 후 종료:
  > 핸드오프 파일이 없습니다 (`.hk/pre-clear/handoff.md`). 이전 세션에서 `/hk:pre-clear:save`를 호출하지 않았거나 이미 resume으로 정리되었습니다. 사용자 메시지로 작업을 시작해주세요.
- **있음** → 다음 단계.

### 2) 핸드오프 파일 읽기

`.hk/pre-clear/handoff.md` 전체를 Read.

### 3) 컨텍스트 흡수 + 사용자 보고

다음 형식으로 사용자에게 한 번에 보고:

```markdown
## 이전 세션 컨텍스트 복원 ✅

**세션 목표:** <한 줄>

**완료한 작업:**

- <항목 요약> (N개)

**진행 중이었던 작업:**
<핵심 1~2 문장>

**다음에 할 일 (우선순위):** (resume 시점에 이미 완료된 항목은 ✅ 표시)

1. <항목>
2. ...

**결정 사항:** <중요한 것 1~3개>

**미해결 질문:** <있으면 — 사용자 결재 필요한 것>

**참조 파일:** <path 나열>

**Suggested Skills / Commands:** <있으면 — 1~3개>
```

### 4) 핸드오프 파일 자동 삭제

복원 보고가 끝난 직후 (사용자 응답 기다리지 않고) 즉시 삭제:

```bash
rm -f .hk/pre-clear/handoff.md
```

### 5) 다음 액션 안내

```
핸드오프 파일은 삭제되었습니다. 위 우선순위대로 진행할까요? 아니면 다른 작업으로 시작하시겠습니까?
```

## 복원 원칙

- **요약하되 누락 금지** — 핸드오프의 모든 결정·미해결 질문은 보고에 포함.
- **민감정보 보호** — handoff에 민감정보가 직접 적혀 있으면 그대로 재출력하지 말고
  redacted 처리 후 사용자에게 파일 정리를 권고한다.
- **참조 파일은 미리 열지 않음** — 사용자가 다음 작업을 지시하면 그때 읽음 (불필요 토큰 소비 방지).
- **삭제는 자동·무확인** — 사용자가 명시적으로 동의한 정책. "삭제할까요?" 같은 확인 질문 금지.
- **읽기 실패 시** — 파일 손상·권한 등으로 읽기 실패하면 사용자에게 보고하고 삭제하지 않음. 사용자가 직접 처리.

## 안티패턴

| ❌                          | 이유                                        |
| --------------------------- | ------------------------------------------- |
| "삭제할까요?" 확인          | 사용자가 자동 삭제 정책 명시. 확인은 노이즈 |
| 핸드오프 전체를 그대로 출력 | 요약 가공이 보고의 핵심. 파일 덤프는 무의미 |
| 참조 파일 자동 Read         | 다음 작업 지시 후에만 열기 — 불필요 토큰    |
| 파일 없음에도 작업 시작     | 사용자에게 상태 알리고 지시 대기            |
| 복원 후 memory에 옮기기     | 진행 중 작업은 memory 정책상 저장 금지      |
